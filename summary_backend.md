# สรุปการพัฒนาระบบ Backend ฝั่งช่าง (Technician Backend Summary)

เอกสารนี้สรุปข้อมูลเกี่ยวกับระบบช่าง (Technician Backend) ที่พัฒนาเสร็จสมบูรณ์แล้ว รวมถึงการเพิ่มระบบจัดการข้อมูลโปรไฟล์ (Profile Management) แบบเต็มรูปแบบเพื่อให้รองรับข้อมูล Database จริง

---

## 1. การปรับปรุงเกี่ยวกับฐานข้อมูลและการแมป Entities
- **ไม่มีการกระทบโค้ดตาราง `Order`**: โครงสร้างคำสั่งซื้อของระบบลูกค้าถูกรักษาไว้ 100% ป้องกันการเกิด Conflict จากทีมพัฒนาตะกร้าสินค้า
- **จัดการข้อมูลการให้บริการ (`TechnicianService`)**: เพิ่มสถาปัตยกรรมในการบันทึกความถนัดของช่างลงตารางกลาง (`technicians_services`) ผ่านคู่เชื่อม `ServiceRepository` และ `TechnicianServiceRepository` เพื่อให้บันทึกเป็น Array IDs [1, 2, 3..] แบบ Many-to-Many
- **อัปเดต Query อย่างตรงจุด**: กำหนดให้ `OrderRepository` ค้นหางานว่าง (PENDING) ให้ตรงกับเขต (Sub-District) ของช่างคนนั้น และค้นหาข้อมูลประวัติการทำงานจาก UUID

---

## 2. โครงสร้าง DTO สำหรับรับ-ส่งข้อมูลฝั่งช่าง (Data Transfer Object)
เพิ่ม DTO ทั้งหมด 4 ตัวเพื่องานช่างเฉพาะทาง:
1. `TechnicianJobDto`: โครงสร้างแสดงข้อมูลบนหน้าการ์ดงาน (Id, Status, Customer Name, Address, Price, Appointment Date, และ Services)
2. `UpdateJobStatusRequest`: สำหรับส่งข้อมูลคำขอเปลี่ยนสถานะงาน (เช่น รับงาน, กำลังดำเนินงาน, เสร็จสิ้น)
3. `TechnicianProfileDto` **(ใหม่!)**: DTO สำหรับส่งข้อมูลโปรไฟล์ไปหน้าตั้งค่าบัญชี (ประกอบด้วย ชื่อ, เบอร์โทร, อีเมล, ที่อยู่, เขตพื้นที่, Toggle พร้อมทำงาน, และ Array ของ Service IDs ความสามารถการซ่อม)
4. `UpdateProfileRequest` **(ใหม่!)**: DTO ขาเข้าเมื่อช่างกดปุ่มเซฟหน้าจอโปรไฟล์ (รับคำขออัปเดตข้อมูลทุกช่อง)

---

## 3. Business Logic หลัก (`TechnicianJobService.java`)
คลาสควบคุมบริการหลัก หรือ **"มันสมองฝั่งช่าง"** ครอบคลุมการทำงานทุกส่วน:
- **`getAvailableJobs()`**: ถอดรหัส Token หาข้อมูลช่าง และค้นหางานว่างที่รอรับในพื้นที่
- **`acceptJob()`**: รับหน้าที่ออเดอร์ เปลี่ยนสถานะเป็น ACCEPTED ประทับตราไอดีช่าง
- **`getMyJobs()`**: ดึงงานส่วนตัวทั้งหมดเพื่อโชว์ในแท็บหน้าประวัติ ทั้งรอดำเนินการและเสร็จแล้ว
- **`updateJobStatus()`**: ปรับเลื่อนสถานะงานซ่อม
- **`getProfile()` & `updateProfile()` (ใหม่!)**: รวบรวมข้อมูลส่วนตัวจาก Entity `User` ตัดคำนามสกุล/ชื่อ สรุปข้อมูลทักษะผ่าน `TechnicianService` เพื่อคืนค่า หรือลบทักษะเก่าทิ้งและ Insert ทักษะใหม่เวลาที่มีการกดอัปเดตออพชันบริการ

---

## 4. API Endpoints (วิธีเรียกใช้งานสำหรับ Frontend API)
หน้าเว็บช่าง (Vue.js) จะสื่อสารกับ Backend ผ่าน Endpoints เหล่านี้ โดยการแนบ **Header** `Authorization: Bearer <token>` เสมอ ข้อมูลเชื่อมผ่านคอนโทรลเลอร์ `TechnicianController.java` ครบวงจรครับ:

| เมธอด (Method) | URL PATH | หน้าที่การทำงานที่เกิดขึ้น |
| :--- | :--- | :--- |
| **GET** | `/api/technician/profile` | ดึงข้อมูลบัญชีโปรไฟล์ปัจจุบันของช่างรวมถึงข้อมูลทักษะ Checkbox หน้า Account Settings |
| **PUT** | `/api/technician/profile` | บันทึกแก้ข้อมูลโปรไฟล์ใหม่ รวมถึงการเปิดปิด Toggle "พร้อมให้บริการ" |
| **GET** | `/api/technician/jobs/available` | ดึงจำนวนงานว่างในพื้นที่ทั้งหมด สำหรับคำนวณป้ายแจ้งเตือนสีแดง (Badge) และโชว์การ์ดใน "คำขอบริการซ่อม" |
| **POST** | `/api/technician/jobs/{orderId}/accept` | กดรับออเดอร์งานนั้นๆ (เปลี่ยนสถานะเป็น ACCEPTED และช่างกลายเป็นเจ้าของ) |
| **GET** | `/api/technician/jobs` | ดึงประวัติรายการซ่อมของตัวเองทั้งหมด (นำไปใช้ Filter ต่อสำหรับ Pending และ History Tasks) |
| **PUT** | `/api/technician/jobs/{orderId}/status` | เลื่อนสถานะงานซ่อมขึ้นลง |

> [!TIP]
> ตอนนี้ฝั่ง Frontend (บริการจัดการที่ `technician.ts`) ผูกรวม API พวกนี้แบบสำเร็จรูปไว้เป็นฟังก์ชันที่เรียกใช้งานง่ายแล้วครับ ช่างแค่เสียบปลั๊ก Login ก็รันได้เลยตลอดทั้งหน้า Dashboard!
